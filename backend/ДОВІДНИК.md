## Бекенд: server.py

> **Роль:** Головний координатор. Тут живуть всі глобальні об'єкти, API-маршрути та фонові завдання.

### Ключові об'єкти (глобальні)

| Змінна | Тип | Призначення |
|--------|-----|-------------|
| `bot_ctx` | `BotContext` | Центральний контекст, "замінник" tkinter app. Зберігає всі DummyVar та посилання на менеджери |
| `STATE` | `dict` | Глобальний стан: поточна ціна, symbol, grids, bot_config |
| `binance_handler` | `BinanceLiveHandler` | Об'єкт для роботи з Binance API |
| `live_manager` | `LiveTradingManager` | Керує запуском/зупинкою реальної торгівлі |
| `market_analyzer` | `MarketAnalyzer` | Технічний аналіз ринку |
| `grid_generator` | `GridGenerator` | Генерація сіток |
| `manager` | `ConnectionManager` | Менеджер WebSocket-підключень фронтенду |

### Клас `BotContext`

Це **не реальний app**, а емулятор `app` для всіх legacy-модулів, написаних під tkinter.

```python
bot_ctx.selected_symbol          # "BTCUSDT"
bot_ctx.live_bot_active          # bool — чи активний лайв-бот
bot_ctx.live_bot_state           # dict: {'orders_data_virtual': [...]}
bot_ctx.live_bot_params          # dict: leverage, initial_group_margin тощо

# DummyVar — замінники tkinter StringVar
bot_ctx.current_price_var.get()  # поточна ціна (str)
bot_ctx.live_long_pos_amt.get()  # кількість монет у LONG позиції (str)
bot_ctx.live_long_pos_entry.get() # середня ціна входу LONG (str)
bot_ctx.live_long_pos_pnl.get()  # PNL рядок "42.5 (2.15 %)"
bot_ctx.live_short_pos_amt.get() # аналогічно для SHORT
bot_ctx.live_short_pos_entry.get()
bot_ctx.live_short_pos_pnl.get()

# Посилання на модулі (прив'язуються після ініціалізації)
bot_ctx.live_trading_manager     # LiveTradingManager
bot_ctx.market_analyzer          # MarketAnalyzer
bot_ctx.grid_generator           # GridGenerator
```

### Клас `MockGuiQueue`

**`bot_ctx.gui_queue`** — замінник tkinter GUI `queue.Queue()`. Замість показу у вікні:
- Логи йдуть у `logger.info`
- WebSocket-події бродкастяться до фронтенду
- Оновлення позицій синхронізуються назад у `bot_ctx.live_long_pos_amt` тощо

**Критично важлива логіка в `MockGuiQueue.put()` при `action == "update_positions_display"`:**
```python
# Коли Binance повертає позиції → оновлюємо DummyVar-и
bot_ctx.live_long_pos_amt.val = str(abs(amt))   # для LONG
bot_ctx.live_long_pos_entry.val = str(entry)
# ...і для SHORT
# Якщо позицію не знайдено → скидаємо в "0"
```

Двосторонній канал з фронтендом.
- **Сервер → Клієнт:** `market_update` (ціна, grids), `gui_event` (позиції, ордери, логи)
- **Клієнт → Сервер:** `update_config` (zміна символу, leverage тощо)

| Задача | Функція | Інтервал |
|--------|---------|----------|

## live_grid_execution_manager.py

> **Роль:** Ядро безпеки та виконання. Аналізує кожен тік — вирішує куди відсунути або коли виставити ордер.

### Клас `LiveGridExecutionManager`

```python
manager = LiveGridExecutionManager(app_instance, binance_api_handler, grid_generator_ref)
```

### Статуси ордерів (константи)

```python
ORDER_STATUS_MONITORING            = "Моніторинг"          # Спостереження, нічого не робимо
ORDER_STATUS_TRIGGERED             = "Тригер"              # Ціна досягла — надсилаємо на біржу
ORDER_STATUS_PLACED                = "Виставлено"          # Ордер на Binance, чекаємо виконання
ORDER_STATUS_REJECTED              = "Відхилено"           # Binance відхилив
ORDER_STATUS_ERROR                 = "Помилка"
ORDER_STATUS_TP                    = "Очікує Тейк-Профіт"
ORDER_STATUS_FAILED_TO_PLACE       = "Помилка виставлення"
ORDER_STATUS_REPOSITIONED_MAJOR    = "Безпека (Погіршення)" # Запарковано — погіршує позицію
ORDER_STATUS_REPOSITIONED_MINOR    = "Безпека (Близько/Вище)" # Запарковано — занадто близько
```

### Ключові параметри безпеки

```python
self.MAJOR_SAFETY_OFFSET_BASE = Decimal("2.5")  # % відступ при погіршенні позиції
self.MAJOR_SAFETY_OFFSET_STEP = Decimal("0.35") # кожен наступний паркований ордер +0.35%
self.MINOR_SAFETY_ZONE_PERCENT = Decimal("1.2") # зона "занадто близько" до входу (%)
self.MINOR_SAFETY_OFFSET_BASE = Decimal("1.5")  # % відступ при minor-порушенні
self.MINOR_SAFETY_OFFSET_STEP = Decimal("0.3")
self.ORDER_PLACEMENT_COOLDOWN_SECONDS = 15      # кулдаун між виставленнями ордерів
```

### Методи

#### `start_managing_grid(orders, params)`
Запускає менеджер. Приймає параметри сітки — symbol, leverage, price_trigger_buffer_percent.

#### `stop_managing_grid(cancel_active_orders=True)`
Зупиняє. За бажанням скасовує всі активні ордери на Binance.

#### `_perform_monitoring_tasks_tick()`
**Головний тік** — викликається кожні 1.5 сек з `live_bot_heartbeat()` у server.py.
1. `_update_real_position_info()` — читає позиції з DummyVar
2. `_synchronize_orders_with_lpb()` — основна логіка

#### `_synchronize_orders_with_lpb()`
**Найважливіший метод.** Логіка:
1. Перевіряє кулдаун (15 сек між ордерами)
2. Зчитує список віртуальних ордерів з `bot_ctx.live_bot_state['orders_data_virtual']`
3. Для кожного ордера визначає чи він **MAJOR** (погіршує середній вхід) або **MINOR** (занадто близько/вище ринку)
4. Якщо -- → пакує, записує в `placed_orders_state` зі статусом `REPOSITIONED_MAJOR/MINOR`
5. Якщо OK → перевіряє тригер (ціна наблизилась на `price_trigger_buffer_percent`)
6. Якщо тригер → виставляє реальний ордер на Binance та активує кулдаун

**Логіка визначення погіршення (Hedge Mode):**
- `LONG/BUY`: Погіршення якщо `order_price > avg_entry_price` (купуємо вище зі середнього)
- `SHORT/SELL`: Погіршення якщо `order_price < avg_entry_price` (продаємо нижче від середнього)

#### `_update_real_position_info()`
Читає `bot_ctx.live_long_pos_amt.get()`, `live_long_pos_entry.get()` і оновлює `self.real_positions_by_side`.

```python
self.real_positions_by_side = {
    "LONG": {"symbol": ..., "positionAmt": Decimal, "entryPrice": Decimal, "positionSide": "LONG"},
    "SHORT": {"symbol": ..., "positionAmt": Decimal, "entryPrice": Decimal, "positionSide": "SHORT"}
}
```

#### `_check_and_manage_tp_levels(pnl_percent, side)`
Перевіряє чи PNL досяг рівнів TP. Якщо так → частково закриває позицію ринковим ордером.

#### `_send_managed_orders_to_gui()`
Формує список для таблиці "Live Managed Orders" у фронтенді. Конвертує всі Decimal → float (важливо для JSON!). Надсилає через `bot_ctx.gui_queue.put(("update_live_managed_grid_orders_display", orders))`.

#### `process_binance_order_update(data)`
Викликається з Binance WebSocket при зміні статусу ордера. `data` — raw Binance order update dict.

#### `process_order_placement_failed(params, error)`
Викликається з `binance_live_handler.py` якщо ордер відхилено.

## binance_live_handler.py

> **Роль:** Всі HTTP-запити до Binance Futures API. Повністю thread-safe через семафор.

### Клас `BinanceLiveHandler`

```python
binance_handler.connected       # bool — чи підключений
binance_handler.client          # binance.Client (None якщо не підключений)
binance_handler.current_position_mode  # True=Hedge, False=One-way
```

### Ключові методи

| Метод | Що робить |
|-------|-----------|
| `connect(api_key, secret)` | Підключається, синхронізує час, перевіряє режим позицій |
| `disconnect()` | Відключається, очищує стан |
| `set_active_symbol(symbol)` | Міняє активний символ, завантажує precisions |
| `fetch_full_position_state(symbol)` | **Головний метод оновлення** — отримує позиції + відкриті ордери разом → надсилає в GUI queue |
| `fetch_open_orders(symbol)` | Завантажує відкриті ордери |
| `place_order(...)` | Розміщує ордер (LIMIT/MARKET/REDUCE_ONLY) |
| `cancel_order(symbol, order_id_str)` | Скасовує конкретний ордер |
| `cancel_all_orders(symbol)` | Скасовує всі ордери |
| `fetch_position_mode()` | Перевіряє One-way vs Hedge mode |

### Потоки запитів

Всі `_task_*` методи виконуються в окремих потоках через `_threaded_api_call()`. Семафор `API_CALL_SEMAPHORE = threading.Semaphore(5)` обмежує 5 одночасних запитів.

## GUI Event System (GUI Queue)

Комунікація між бекендом і фронтендом відбувається через систему подій.

**На бекенді:**
```python
bot_ctx.gui_queue.put(("event_name", data))
```

**`MockGuiQueue.put()` в server.py:**
1. Логує події (окрім частих "шумових")
2. Спеціальна обробка для `"update_positions_display"` → синхронізує DummyVar-и позицій
3. Бродкастить до фронтенду через WebSocket:
   ```python
   manager.broadcast_message({"type": "gui_event", "action": event_name, "data": data})
   ```

### Таблиця GUI-подій

| Подія (`action`) | Дані | Що робить фронтенд |
|-----------------|------|-------------------|
| `"live_log"` | `str` | Додає рядок у лог-панель |
| `"update_positions_display"` | `{"positions": [...]}` | Оновлює таблицю позицій |
| `"update_open_orders_display"` | `[...]` | Оновлює таблицю відкритих ордерів на Binance |
| `"update_live_managed_grid_orders_display"` | `[ManagedOrder]` | Оновлює таблицю "Live Managed Orders" |
| `"update_trade_history_display"` | `[...]` | Оновлює таблицю торгової історії |
| `"update_connection_status"` | `{"connected": bool, "message": str}` | Статус підключення |
| `"live_trading_status_update_button"` | `{"active": bool}` | Статус лайв-боту |
| `"live_bot_status_update_display"` | `{"main_status": ..., "grid_status": ..., "pnl_status": ..., "action_status": ...}` | Статус-рядки бота |
| `"symbol_data_status"` | `{"ready": bool, "status_message": str}` | Готовність даних символу |

### Ключові HTTP-виклики з фронтенду

```typescript

handleApiRequest("connect")          // POST /api/connect
```

## Поширені помилки та рішення

| Проблема | Причина | Рішення |
|----------|---------|---------|
| `NameError: lpb_orders` | Змінна не оголошена | Перевірити `_send_managed_orders_to_gui` — `lpb_orders = self.app.live_bot_state.get(...)` |
| `JSON serialization error` | Decimal не конвертований у float | Всі дані для GUI мають бути `float(Decimal(...))` |
| Бот не паркує ордери | `real_positions_by_side` пустий | Перевірити `MockGuiQueue.put` → чи оновлюються `live_long_pos_amt.val` |
| Позиції не скидаються | Відсутній `if not found_sides["LONG"]: reset...` | Виправлено в останній версії server.py |
| Правила TP не активні | `load_live_trading_rules()` не викликано | Викликається на старті сервера після ініціалізації `live_manager` |

## Важливі правила при розробці

1. **Всі числа для GUI = `float`**. `Decimal` не серіалізується в JSON. Завжди `float(Decimal(...))` в `_send_managed_orders_to_gui`.
2. **Hedge Mode завжди**: `BUY` → `positionSide=LONG`, `SELL` → `positionSide=SHORT`. Ніякого `BOTH`.
3. **DummyVar читається через `.get()`**: `bot_ctx.live_long_pos_amt.get()` → `str`. Конвертувати в Decimal перед використанням.
4. **`placed_orders_state`** — внутрішній словник менеджера, `key = order_id_sim (v_id)`.
5. **Не змінювати `bot_ctx.after()` та `bot_ctx.after_cancel()`** — це заглушки для сумісності з tkinter.
